# 01 系统总览

## 目标

构建一个“用户给素材与场景描述，系统自动生成像素风场景”的平台能力，输出可运行、可编辑、可验证的工程产物。

### 输入
- 像素 tileset / placeable 对象 / 角色 spritesheet
- 场景文字描述
- 可选的玩法约束（是否有角色移动、是否有碰撞、是否有天气）
- 可选的风格参考图和布局偏好

### 输出
- Phaser 运行时工程
- Tiled 地图工程
- Scene Spec / Layout Plan / Generation Report
- 预览截图与校验结果

## 非目标

- 不做通用 3D 世界生成
- 不在第一阶段支持开放世界或复杂任务脚本
- 不让 LLM 直接生成底层 tile index 矩阵
- 不把 Tiled CLI 当作完整地图编辑 API

## 关键角色

### 最终用户
上传素材，描述场景，给出修改意见。

### 内容设计者
维护命名规范、对象类别、可放置规则、天气预设、Tiled custom classes。

### Agent 编排器
把用户意图拆成多个任务，安排素材理解、规划、构建、验证、修复。

### Map Builder
根据结构化规格生成地图层、对象层、导航网格、碰撞层。

### Phaser Runtime
最终渲染和运行场景，承载玩家移动、摄像机跟随、天气、交互。

## 成功标准

1. 用户一句话能生成一张可运行像素地图
2. 生成后能在 Tiled 中继续编辑
3. 角色可移动，相机跟随正常
4. 天气效果参数化，不写死在单个场景里
5. 任何一次生成都有可追溯的 spec / report / preview
6. 用户追加要求时，系统优先增量修改而不是整体重建

## 典型用户请求

> 做一个像素公园。中央是 Y 字形主路。左上角一片树林，右下角一个小咖啡亭。沿路放 10 张长椅，角色从中心出生，天气是小雨。

系统应当把这段描述转换为：

- `scene-spec.json`
- `layout-plan.json`
- `tiled/maps/main.tmx`
- `tiled/maps/main.export.json`
- `generation-report.json`
- `preview.png`（后续可补）

## 系统设计主线

1. 素材接入与标注
2. Scene Spec 生成
3. 布局规划
4. Tiled 工程落地
5. Phaser 运行时绑定
6. 自动验证与修复
7. 迭代变更与补丁
